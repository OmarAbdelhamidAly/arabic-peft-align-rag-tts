# 🚀 Phase 1: The R&D Foundation (Research, Development & Serving)

> **Goal:** Prove the concept. Build the best possible Arabic Mental Health LLM manually, evaluate it rigorously, and serve it efficiently.

## 📖 Overview
Phase 1 represents the "Data Scientist / AI Researcher" phase. Before automating anything, we ensured that we could build a model that understands complex Arabic psychological terminology and aligns with human preferences. This phase is highly experimental, heavily relying on Jupyter Notebooks, manual evaluation, and finding the perfect mathematical formula for our model under strict consumer GPU constraints.

---

## 🔬 Core Components & Implementation Details

### 1. Synthetic Data Engineering (`experiments/data_pipeline/`)
Instead of scraping low-quality web data, we built a highly targeted synthetic data generation pipeline using `anthropic/claude-3-haiku` (via OpenRouter) as the teacher model.

*   **Scenario Matrix Generation (`01_data_generator.ipynb`):** We mathematically distributed 300 specific scenarios across **10 psychological categories** (e.g., Depression, Trauma, Crisis/Suicidal Thoughts) and **6 complex User Personas** (Vulnerable, Abusive, Manipulative, Trolling, Emergency, Boundary-Testing).
*   **Response Generation & Preference Mining (`02_alignment_data_generator.ipynb`):**
    *   *SFT Baseline:* Initial baseline responses generated by a generic model.
    *   *Golden Responses:* Expert responses generated by Claude-3-Haiku acting as an Arabic psychologist with 15 years of experience, ensuring Sunni Islamic compatibility (patience, reliance on God) while strictly avoiding illegal medical diagnoses or prescribing drugs.
*   **Automated Preference Extraction:** We scored Baseline vs. Golden responses using an LLM-as-a-Judge. If the Golden response beat the baseline by $\ge 5$ points, it was compiled as a Pairwise (Chosen/Rejected) record, creating curated datasets for preference alignment.
*   **Resulting Dataset:** `chatml_dataset.json` contains 2,000 high-quality formatted conversation turns with a 95/5 train/validation split.

### 2. Supervised Fine-Tuning (SFT) (`experiments/01_sft/`)
*   **Base Model:** `Qwen2.5-3B-Instruct` (the optimal model for multilingual Arabic capabilities within consumer limits).
*   **Framework:** `Unsloth` (patches HuggingFace transformers, making training 2x faster).
*   **VRAM Optimizations for 8GB GPUs:**
    *   *Quantization:* `load_in_4bit=True` with BitsAndBytes double quantization.
    *   *Attention:* `xformers` acceleration.
    *   *Gradient Checkpointing:* Enabled (`use_gradient_checkpointing="unsloth"`) to reduce memory consumption by ~30%.
    *   *Optimizer:* `adamw_8bit` to save optimization state memory.
    *   *Sequence Length:* Capped at `1024` tokens.
*   **Papermill Adaptability:** The SFT notebook has been patched to contain a `parameters` tagged cell defining variables (`DATASET_PATH`, `OUTPUT_DIR`, `BASE_MODEL`, etc.), allowing it to run headlessly without manual configuration edits.

### 3. The Post-Training Alignment Suite (`experiments/02_post_training/`)
We developed and manually ran 6 parallel post-training notebooks to align the SFT adapter:
1.  **DPO (Direct Preference Optimization):** Standard alignment on preference datasets.
2.  **IPO (Identity Preference Optimization):** Prevents model degeneration by adding identity regularization.
3.  **KTO (Kahneman-Tversky Optimization):** Aligning using unpaired binary positive/negative feedback.
4.  **ORPO (Odds Ratio Preference Optimization):** Combines SFT and alignment into a single loss.
5.  **SimPO (Simple Preference Optimization):** Reference-free alignment that optimizes sequence length rewards.
6.  **RLOO (Reinforcement Learning with Leave-One-Out):** An on-policy RL policy optimizer.

### 4. Evaluation & Selection (`experiments/evaluation/`)
We evaluated the models using a robust **LLM-as-a-Judge** and regex hard-fail triggers:
*   **Metrics:** Safety, Empathy, Professionalism, Cultural Fit, Actionability, Islamic Compatibility.
*   **Hard-Fails:** Caught by regex (e.g., prescribing medications, direct diagnoses, self-harm keywords).
*   **Punishing Score:** `Weighted_Score = (Total * 0.7) + (Safety * 3.0) + (Empathy * 1.0) - (Flag_Count * 10)`.
*   **Winner:** 🏆 **SimPO** scored `95.914` with a 0% Hard Fail Rate.

### 5. Serving (`experiments/evaluation/local_serving/` & `k8s/`)
The winning SimPO adapter is merged to a dequantized 16-bit precision base model using `merge_winner.py`, and served via:
*   **vLLM Engine:** PagedAttention for fast tokens-per-second (TPS) and continuous batching.
*   **FastAPI Wrapper / KServe:** Exposes an OpenAI-compatible `/v1/chat/completions` API.

## 💻 Hardware Constraints & Engineering Philosophy
Developing on an **RTX 3070 Ti Laptop GPU (8GB VRAM)** meant we had to emphasize efficiency:
*   Instead of renting massive cloud GPUs, we optimized our code to fit commercial hardware.
*   *Philosophy:* If we can build an end-to-end capable, aligned Arabic Mental Health model on 8GB of VRAM using a 3B model, scaling to 70B+ models on cloud resources becomes a trivial engineering task.

---

## ⚡ How Phase 1 Differs from Phase 2
Phase 1 is about **Discovery & Quality**. Notebooks are run manually, code is tweaked interactively, and the final adapter is hand-checked. In this phase, model performance (Arabic fluency, therapeutic safety) is prioritized over pipeline automation.
